Point d’entrée

Le parcours va du vocabulaire de base vers la CLI et les erreurs stables. Pour débuter vite, lire `docs/book/poche/SOMMAIRE.md`, puis revenir ici pour les repères courts.

Comment lire le book

Cette page est la table de matières complète du livre Vitte. Elle sert à trois usages: démarrer vite, retrouver un chapitre précis, ou traverser le plan entier du projet sans perdre le fil entre le langage, la stdlib, la CLI et la qualité documentaire.

  • Débuter : ouvrir le sommaire poche, puis les chapitres 01, 04, 05 et 10.
  • Comprendre : suivre les chapitres 09, 15, 17, 27 et 32 pour le cœur du projet.
  • Construire : lire les chapitres 41, 58, 69 à 74 pour les pratiques avancées.
  • Corriger : passer par les chapitres 10, 31, 35, 47 et la page diagnostics du site.

Mode d'emploi

Écrire

Favoriser des blocs courts, des noms explicites, et des imports canoniques.

Vérifier

Relire le premier diagnostic, puis utiliser `vitte explain E0001` si besoin.

Itérer

Corriger, relancer, puis comparer la sortie avec `--diag-code-only` quand c'est utile.
