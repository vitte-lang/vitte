Dépôt

Le dépôt est présenté sous forme de repères courts pour garder une lecture rapide et relier les pages docs, diagnostics, contribution, et book.

Comprendre le compilateur

Le compilateur Vitte se lit comme une suite de couches simples: parser, résolution, IR, puis backend. Chaque étape produit une sortie plus structurée, et chaque erreur doit pouvoir se rattacher à une couche précise.

Parser forme le code et construit l'AST
→
Resolve relie noms, imports et symboles
→
IR abaisse vers une représentation intermédiaire
→
Backend produit le code cible

Références: pipeline compilateur, architecture globale, mapping AST -> IR.
