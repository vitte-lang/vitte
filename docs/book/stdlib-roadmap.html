Vitte Stdlib Roadmap

This document describes the target organization in Vitte Stdlib. Mandatory terminology: we are talking about stdlib (not crates).

Import convention in language:

  • stdlib name: vitte-core (documentation/architecture).
  • path in use: vitte/core (Vitte syntax).

1) Language foundation

vitte-core

  • primitive types.
  • error management (Result, Option).
  • basic traits.
  • abstract allocation.

vitte-alloc

  • global allocator.
  • bump/arena.
  • slab.
  • fallback no_std.

vitte-collections

  • Vec.
  • Map.
  • Set.
  • RingBuffer.
  • Rope (useful for PlatonEditor).

vitte-math

  • int / float.
  • digital traits.
  • random.
  • SIMD wrapper.

19) Strategic orientation

If Vitte aims for system language, priority:

  • alloc.
  • async.
  • net.
  • fs.
  • abi.
  • codegen.

If Vitte aims for a modern backend, priority:

  • web.
  • db.
  • auth.
  • json.
  • async runtime.

If Vitte aims for tooling / IDE, priority:

  • rope.
  • lsp.
  • syntax.
  • incremental parser.

If Vitte aims for data/scientific, priority:

  • array.array.
  • stats.
  • plot.
  • ml.

Architectural rules

  • vitte-core does not depend on any high-level module.
  • vitte-alloc depends on vitte-core.
  • vitte-collections depends on vitte-core + vitte-alloc.
  • vitte-std re-export; the heavy logic remains in the specialized stdlib modules.
  • vitte-runtime depends on vitte-async + vitte-net + vitte-io.
  • vitte-ffi applies the contracts of vitte-abi.

MVP priority (recommended order)

  1. vitte-core.
  2. vitte-alloc.
  3. vitte-collections.
  4. vitte-io.
  5. vitte-async.
  6. vitte-runtime.
  7. vitte-net.
  8. vitte-fs / vitte-process.
  9. vitte-std (stable re-exports).
