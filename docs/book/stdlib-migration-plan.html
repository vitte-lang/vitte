Stdlib Migration Plan

Goal: move from historical use std/* towards a coherent stdlib/* space, with progressive compatibility.

1) Target conventions

  • Required module entry: mod.vit
  • Required module metadata: info.vit
  • Public names without _ (prefer hierarchy with /)
  • std/* kept only as legacy facade (re-export)

2) Existing Legacy (frozen, no new cases)

Active legacy wrappers (historical path import):

  • abi/abi.vit
  • actor/actor.vit
  • alerts/alerts.vit
  • alloc/alloc.vit
  • array/array.vit
  • ast/ast.vit
  • async/async.vit
  • audio/audio.vit

Legacy names with _:

  • circuit_breaker
  • config_derive
  • const_eval
  • feature_flags
  • http_client
  • mock_http
  • random_secure
  • rate_limit
  • schema_registry

3) Migration table old -> new

  • http_client -> http/client
  • mock_http -> http/mock
  • random_secure -> random/secure
  • schema_registry -> schema/registry
  • rate_limit -> rate/limit
  • circuit_breaker -> resilience/circuit_breaker
  • feature_flags -> config/feature_flags
  • const_eval -> compiler/const_eval
  • config_derive -> config/derive

Duplicates/clarifications:

  • process vs subprocess: keep process public API and move the low level under process/internal (or alias subprocess -> process/spawn).
  • runtime vs core: core = stable primitives, runtime = bootstrap/runtime.
  • serialize vs codec: codec = concrete encoding (json/yaml/bin), serialize = stable facade.

4) Target families

  • core/*: types, collections, alloc, borrow
  • system/*: fs, env, process, net, path
  • data/*: json, yaml, sql, db, cache, stream
  • app/*: http, ws, cli, config, jobs

5) Compatibility and migration

  • Phase A: create the new stdlib modules + re-exports from the old ones.
  • Phase B: depreciation warnings on legacy imports.
  • Phase C: auto-fix (vitte mod doctor --fix) old -> new.
  • Phase D: gradual removal of legacy aliases after a compat window.

Plan C2 (proposed):

  • Add vitte mod doctor --fix --write <file.vit> to apply rewrites directly to the file.
  • Keep --fix without --write in dry-run mode (text suggestions only).
  • Add a mode --fix --write --backup to create <file>.bak before writing.
  • Prioritize safe and syntactic rewrites (vitte/x/x -> vitte/x) before any more intrusive transformation.

Depreciation window:

  • v2.x: compat active + warning E1020 on vitte/x/x.
  • v3.0: removal of the compat vitte/x/x (canonical import required vitte/x).

6) CI

Active rule: make package-layout-lint.

This rule:

  • refuse any new stdlib module without mod.vit/info.vit;
  • refuse any new name with _;
  • keeps a legacy allowlist for the existing one.

Allowlist files:

  • tools/package_entry_legacy_allowlist.txt
  • tools/package_name_legacy_allowlist.txt
