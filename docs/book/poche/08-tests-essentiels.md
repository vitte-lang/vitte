# 8. Tester sans se perdre

Prérequis: `docs/book/poche/07-lire-les-erreurs.md`.
Voir aussi: `docs/book/chapters/18-tests.md`.

## Objectif

Écrire des tests courts qui sécurisent les bases.

## Exemple

Cas nominal + cas limite pour chaque procédure critique.

## Pourquoi

Les tests évitent les régressions pendant les refactors.

## Test mental

Quel est le plus petit test utile pour `add(a, b)`?

## À faire

1. Ajouter un test nominal.
2. Ajouter un test limite.
3. Vérifier le résultat attendu.

## Corrigé minimal

2 tests bien ciblés valent mieux que 20 tests redondants.
