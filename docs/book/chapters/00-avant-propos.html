History of the Project

Vitte was created with the objective of providing a language that remains understandable from small utilities to large software systems. The project began as an exploration of compiler construction, software architecture, and long-term maintainability.

Many existing languages provide powerful features but often accumulate historical complexity over decades of evolution. Vitte attempts to start from a clean foundation while preserving practical software engineering requirements.

The language is developed alongside its compiler, standard library, tooling ecosystem, documentation, diagnostics framework, package manager, and editor integrations.

Project Goals

The primary goals of the project are:

  • Predictable compilation.
  • Clear diagnostics.
  • Strong tooling integration.
  • Readable syntax.
  • Explicit ownership of software architecture.
  • Long-term maintainability.
  • Reproducible builds.
  • Scalable project organization.
  • Cross-platform development.
  • Practical compiler implementation.

Non Goals

Understanding what a language does not attempt to solve is just as important as understanding what it does solve.

  • Vitte does not attempt to minimize every line of code.
  • Vitte does not prioritize clever syntax tricks.
  • Vitte does not encourage hidden runtime behavior.
  • Vitte does not optimize for code golfing.
  • Vitte does not treat readability as optional.

Language Philosophy

The language follows a simple principle: software is read more often than it is written.

As a consequence, source code should communicate intent as clearly as possible to future maintainers.

A Vitte source file should remain understandable months or years after its creation.

This philosophy influences naming conventions, diagnostics, module organization, compiler architecture, and standard library design.

The Role of Types

Types serve as executable documentation. A well-designed type system communicates assumptions, constraints, and expected behavior directly through source code.

Instead of relying exclusively on comments, Vitte encourages developers to express intent through forms, picks, procedure signatures, and module boundaries.

The Role of Diagnostics

Diagnostics are a fundamental component of the language experience. A compiler should not merely reject invalid programs. It should explain why a program is invalid and provide actionable information for resolving the issue.

Every diagnostic produced by the compiler is expected to contain structured information including severity, location, context, and recommendations when appropriate.

Documentation Philosophy

This documentation is intentionally verbose. The objective is not only to teach syntax but also to explain the reasoning behind language decisions.

Whenever possible, examples are accompanied by explanations, trade-offs, limitations, and common mistakes.

Readers should understand both how a feature works and why it exists.

Compiler Overview

The Vitte compiler is divided into multiple stages. Each stage transforms the program into a progressively more structured representation.

  1. Source loading.
  2. Lexical analysis.
  3. Parsing.
  4. Abstract Syntax Tree generation.
  5. High-Level Intermediate Representation.
  6. Semantic analysis.
  7. Type checking.
  8. Borrow checking.
  9. Mid-Level Intermediate Representation.
  10. Backend lowering.
  11. Object generation.
  12. Linking.

Understanding these stages is not required to write programs, but it helps explain compiler diagnostics and performance behavior.

Standard Library Overview

The standard library provides foundational functionality for application development.

  • Algorithms.
  • Collections.
  • Memory utilities.
  • Filesystem support.
  • Networking.
  • Serialization.
  • Concurrency primitives.
  • Mathematics.
  • Diagnostics helpers.
  • Testing infrastructure.

Tooling Ecosystem

A programming language is more than a compiler. The ecosystem includes editors, language servers, formatters, package management tools, documentation generators, testing frameworks, profiling tools, and build systems.

The Vitte ecosystem is designed to provide consistent workflows across development environments.

Versioning and Stability

Language evolution must balance innovation and stability. Breaking changes should be carefully evaluated, documented, and justified.

Backward compatibility is considered whenever possible, particularly for stable releases.

Who Should Read This Book

  • Developers discovering Vitte.
  • Students learning compiler concepts.
  • Library authors.
  • Tooling developers.
  • Systems programmers.
  • Application developers.
  • Compiler contributors.

Expected Knowledge

No prior knowledge of Vitte is required. Basic familiarity with programming concepts such as variables, functions, conditions, and data structures is sufficient.

Advanced compiler knowledge is not required. Relevant concepts are introduced progressively throughout the book.

Conventions Used In This Documentation

  • Source code appears in dedicated code blocks.
  • Compiler diagnostics are shown verbatim when possible.
  • Important concepts are introduced before advanced usage.
  • Examples prioritize readability over brevity.
  • Cross-references connect related chapters.

Final Words Before Starting

The goal of this documentation is not merely to teach syntax. Its purpose is to help readers understand how to design, maintain, debug, and evolve software using Vitte.

Take the time to read the examples carefully, pay attention to diagnostics, and focus on the reasoning behind each construct.

The chapters that follow build upon these principles and gradually introduce the complete language and ecosystem.
