Keyword when

Level: Intermediate.

Quick reading

Concrete reference: when is used to decide an execution path in a readable and verifiable manner. Use when when it is used to drive a control decision; avoid it if it makes no observable difference on the branch or output.

Why (job)

In practice, when should make the rule clearer, not noisier. If removing it changes neither the meaning nor the outcome, prefer the simpler form.

Definition

when occurs in select and in the form when expr is pattern.

Syntax

Canonical form: when pattern block or when expr is pattern block.

Algorithmic reading

Read it in context: find where when appears, then confirm the constraint it adds.

Nominal example

Input:

  • Form when expr is pattern.
// Exemple concret: cas nominal puis cas invalide

pick Resp { case Ok, case Err }

proc on_resp(r: Resp) -> int {
  when r is Ok { give 0 }
  // Valeur retournee par cette branche
  give 1
}

Observed result:

  • The when branch is activated only if the pattern matches.

Invalid example

Input:

  • Incomplete clause.
// Exemple concret: cas nominal puis cas invalide
when r is { give 0 }
# invalide: pattern manquant après `is`.

Observed result:

  • The parser rejects the clause.

Expected diagnosis:

  • Code: VITTE-XXXX (or corresponding E000X compiler code).
  • Position: ligne 1, colonne 1 (adjust according to actual snippet).
  • Message: stable, actionable correction.

Quick refactor

Before:

// Exemple concret: cas nominal puis cas invalide
# usage fragile à corriger

After:

// Exemple concret: cas nominal puis cas invalide
# usage clair et testable

Pitfalls

  • Write when without a block.
  • Put an invalid literal as pattern.
  • Confuse when and case.

When to use it / When to avoid it

  • When to use it: to express a branch based on a pattern.
  • When to avoid it: for a simple direct Boolean test (if).
