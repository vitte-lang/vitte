Keyword use

Level: Intermediate.

Quick reading

Concrete reference: use is used to decide an execution path in a readable and verifiable manner. Use use when used to organize modules and symbols; avoid it if it makes no observable difference on the branch or output.

Why (job)

In algorithmic practice, use is used to transform a condition into an executable branch, without reading ambiguity. Rule of thumb: if removing use changes neither the executed path nor the output, simplify the block.

Definition

use imports symbols from a path (glob, group or alias).

Syntax

Canonical form: use path[.{...}|.*] [as alias].

Algorithmic reading

Recommended Reading: Track down where use occurs in the flow, then check its effect on actual execution.

Nominal example

Entrance:

  • Import top-level group.
// Exemple concret: cas nominal puis cas invalide

space app/core
use std.io.{read, write} as io

proc ping() -> int {
  // Valeur retournee par cette branche
  give 0
}

Observable output:

  • Imported symbols are visible via the alias io.

Invalid example

Entrance:

  • use placed in a procedure.
// Exemple concret: cas nominal puis cas invalide

proc bad() -> int {
  use std/bridge/io
  // Valeur retournee par cette branche
  give 0
}
# invalide: `use` est top-level.

Observable output:

  • The compiler rejects the declaration position.

Expected diagnosis:

  • Code: VITTE-XXXX (or corresponding E000X compiler code).
  • Position: ligne 1, colonne 1 (adjust according to actual snippet).
  • Message: stable pattern oriented correction.

Quick refactor

Before:

// Exemple concret: cas nominal puis cas invalide
# usage fragile à corriger

After:

// Exemple concret: cas nominal puis cas invalide
# usage clair et testable

Pitfalls

  • Import * everywhere without necessity.
  • Multiply opaque aliases.
  • Leave dead imports.

When to use it / When to avoid it

  • When to use it: to import an explicit subset of symbols.
  • When to avoid it: in a block or with an uncontrolled glob.
