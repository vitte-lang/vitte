Keyword make

Level: Beginner.

Quick reading

Concrete reference: make is used to decide an execution path in a readable and verifiable manner. Use make when it is used to structure the execution flow; avoid it if it makes no observable difference on the branch or output.

Why (job)

In practice, make should make the rule clearer, not noisier. If removing it changes neither the meaning nor the outcome, prefer the simpler form.

Definition

make declares a variable (global or local) with optional type via as.

Syntax

Canonical form: make nom [as type] = expr.

Algorithmic reading

Read it in context: find where make appears, then confirm the constraint it adds.

Nominal example

Input:

  • Valid local initialization.
// Exemple concret: cas nominal puis cas invalide

proc init() -> int {
  make counter as int = 0
  // Valeur retournee par cette branche
  give counter
}

Observed result:

  • counter is initialized then returned.

Invalid example

Input:

  • Use of make as an expression.
// Exemple concret: cas nominal puis cas invalide

proc bad() -> int {
  let c: int = make 0
  // Valeur retournee par cette branche
  give c
}
# invalide: `make` est une déclaration, pas une expression.

Observed result:

  • The parser rejects the instruction.

Expected diagnosis:

  • Code: VITTE-XXXX (or corresponding E000X compiler code).
  • Position: ligne 1, colonne 1 (adjust according to actual snippet).
  • Message: stable, actionable correction.

Quick refactor

Before:

// Exemple concret: cas nominal puis cas invalide
# usage fragile à corriger

After:

// Exemple concret: cas nominal puis cas invalide
# usage clair et testable

Pitfalls

  • Confusing make and value constructor.
  • Omit = from the declaration.
  • Mix let/make conventions in the same block without a rule.

When to use it / When to avoid it

  • When to use it: to explain the creation of a variable with a type constraint.
  • When to avoid it: as expression (make is not a literal).
