Keyword is

Level: Intermediate.

Quick reading

Concrete reference: is is used to decide an execution path in a readable and verifiable manner. Use is when it is used to drive a control decision; avoid it if it makes no observable difference on the branch or output.

Why (job)

In practice, is should make the rule clearer, not noisier. If removing it changes neither the meaning nor the outcome, prefer the simpler form.

Definition

is associates an expression with an pattern (cast/pattern test in cast_expr or when ... is ...).

Syntax

Canonical form: expr is pattern.

Algorithmic reading

Read it in context: find where is appears, then confirm the constraint it adds.

Nominal example

Input:

  • Pattern check on variant.
// Exemple concret: cas nominal puis cas invalide

pick Resp { case Ok, case Err }

proc is_ok(r: Resp) -> bool {
  // Valeur retournee par cette branche
  give r is Ok
}

Observed result:

  • Returns true if r matches the pattern Ok.

Invalid example

Input:

  • Right operand absent.
// Exemple concret: cas nominal puis cas invalide

proc bad(r: int) -> bool {
  // Valeur retournee par cette branche
  give r is
}
# invalide: pattern manquant.

Observed result:

  • The parser rejects the expression.

Expected diagnosis:

  • Code: VITTE-XXXX (or corresponding E000X compiler code).
  • Position: ligne 1, colonne 1 (adjust according to actual snippet).
  • Message: stable, actionable correction.

Quick refactor

Before:

// Exemple concret: cas nominal puis cas invalide
# usage fragile à corriger

After:

// Exemple concret: cas nominal puis cas invalide
# usage clair et testable

Pitfalls

  • Use is to replace == everywhere.
  • Use an undeclared pattern.
  • Forgetting full flow coverage after testing.

When to use it / When to avoid it

  • When to use it: to test a form of value via pattern.
  • When to avoid it: for a simple scalar equality test (==).
