Compilateur Vitte

La vue source lit le projet via la pile du compilateur Vitte : la surface du pilote, la surface grammaticale, les instantanés et les vérifications de dépôt sont alignés autour des mêmes chemins de code.

Comprendre le compilateur

Le compilateur Vitte se lit comme une série de couches simples : analyseur, résolution, IR, puis backend, avec le pilote et la surface de peluchage exprimés dans le même langage que le reste du référentiel.

  • Analyseur : lit le code et construit l'AST.
  • Résolution : relie les noms, les importations et les symboles.
  • IR : descend à une représentation intermédiaire.
  • Back-end : produit le code cible.

Couler

  • Flux principal : source -> parser -> AST -> resolve -> IR -> backend -> output
  • Erreurs de lecture : diagnostic -> layer -> cause -> minimal fix -> rerun

Notes actuelles du compilateur

  • La mise en œuvre du pilote réside sous src/vitte/compiler/driver.
  • Le travail de synchronisation de la grammaire et de l'analyseur pointe désormais vers src/vitte/grammar/vitte.ebnf, src/vitte/grammar/vitte.pestet le rapport de couverture de l'analyseur.
  • Les vérifications au niveau du dépôt telles que la disposition des packages, les lints d'importation hérités et les vérifications du chemin des documents sont adaptées à l'arborescence Vitte uniquement.
