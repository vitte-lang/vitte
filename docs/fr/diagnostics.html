Vitte Diagnostics

Consultation rapide des diagnostics : un code, une couche, une correction.

Source de vérité

Commencez par le premier code, puis lisez le fichier stable correspondant dans docs/book/errors.html. Pour confirmer la signification du message, parcourez docs/book/cli.html et la commande vitte explain <code>.

Séquence officielle

  • vitte explain <code>
  • vitte check --diag-code-only
  • vitte mod doctor --fix
  • vitte check --diag-json-pretty

Codes fréquents

  • E0001 - identifiant attendu
  • E0013 - identifiant inconnu
  • E0019 - la procédure nécessite un corps
  • E0024 - la sélection en nécessite au moins un lors de la branche

Que faire en premier

  • Lisez le premier code, pas toute la cascade.
  • Ouvrez le fichier correspondant dans docs/book/errors.html.
  • Lancer vitte explain <code> pour confirmer le sens officiel.
  • Corrigez localement, puis redémarrez sur un cas minimal.

Index rapide

  • analyser : E0001 à E0008.
  • résoudre : E0009 à E0025.
  • IR : E0015 à E0028.

Lecture pratique

Les diagnostics doivent être lus comme un contrat et non comme un mur de texte. Le code indique ce qui a échoué, le livre indique ce que signifie le code et la CLI explique comment l'inspecter. Si ces trois éléments ne sont pas d'accord, le premier endroit auquel faire confiance est la source actuelle, puis le chapitre dédié, puis le résultat de la commande.

Pour un tri rapide, utilisez cette commande :

  1. Identifiez le code.
  2. Ouvrez l'entrée correspondante dans docs/book/errors.html.
  3. Courir vitte explain <code> pour confirmer le texte actuel.
  4. Réduisez la reproduction au plus petit extrait qui échoue toujours.
  5. Vérifiez si le problème appartient à l'analyse, à la résolution, à la réduction ou à une règle de post-analyse.

Cette séquence aide les contributeurs à rester précis. Cela rend également les rapports de bogues utiles, car le rapport nomme le code, la couche, la plus petite reproduction et le comportement attendu au lieu de décrire uniquement le symptôme.

Si vous avez besoin d'un point de départ plus large, lisez Documentation pour la surface du compilateur et Source pour le chemin de mise en œuvre.
