Règle simple : une seule action principale par bloc, puis on affine seulement si le contexte l'exige.

Comment choisir

Lorsque le projet vous semble chargé, choisissez le plus petit bloc qui fait avancer le travail. Si vous avez besoin de comprendre une fonctionnalité, commencez par Documentation. Si vous avez besoin de prouver le comportement, rendez-vous sur Diagnostic. Si vous avez besoin de l'implémentation actuelle, ouvrez Source. Si vous avez besoin du dernier contexte du projet, lisez Nouvelles. La page de suggestions existe pour que la prochaine action reste évidente.

L'ordre général est stable :

  1. Lisez la page qui décrit la règle.
  2. Vérifiez l'implémentation ou le chemin d'erreur qui l'applique.
  3. Exécutez le plus petit test pouvant confirmer le comportement.
  4. Ensuite seulement, élargissez la modification aux documents, exemples ou notes de version.

Cela rend le travail facile à réviser et à expliquer. Cela permet également de garantir l'honnêteté du référentiel : chaque suggestion doit pointer vers un artefact concret, et pas seulement vers une vague intention.
