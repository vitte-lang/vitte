Fixer des objectifs

Cette page répertorie les plus utiles make points d'entrée.

Pour la liste complète, exécutez :

make help

Débutant / Boucle locale

  • make build: tout
  • make quickstart-check : vérifiez le chemin du débutant par rapport à examples/first_project.vit
  • make doctor : imprimer la chaîne d'outils locale et la préparation à l'environnement
  • make test: exécuter des tests (std/test)
  • make parse : exécuter des tests au niveau de l'analyseur
  • make hir-validate : exécuter les appareils de test du validateur HIR

Langue de base

  • make grammar-check : échoue si les artefacts générés par la grammaire sont désynchronisés
  • make grammar-gate: exécutez la vérification grammaticale + le test grammatical
  • make core-language-gate : exécutez la vérification grammaticale + le test de langue de base + les portes sémantiques principales + les paramètres régionaux de diagnostic lint
  • make core-release-gate : exécutez la porte des contrats en langue protégée pour les travaux liés aux versions

Modules / Stdlib

  • make modules-tests: exécuter le module graph/doctor luminaires
  • make modules-snapshots: affirmer les sorties du graphique du mod/du médecin
  • make modules-contract-snapshots : asservir des instantanés de contrat de modules
  • make packages-gate : exécutez la gouvernance du package/la mise en page/les performances/la porte du contrat

CI / Rapports

  • make ci-fast : exécute une chaîne CI rapide pour les instantanés et les vérifications du langage/du compilateur
  • make ci-strict : exécutez une chaîne CI stricte avec des diagnostics/vérifications de mise en évidence supplémentaires
  • make ci-completions: exécuter la vérification d'achèvement + peluches + instantanés + secours
  • make repo-hygiene-check : vérifier les contraintes d'hygiène du référentiel au niveau racine
  • make docs-paths-check : vérifiez que les références de chemin local README/docs existent
  • make dx-adoption : exécuter les portes de qualité DX/adoption
  • make public-benchmark-dashboard: générer un tableau de bord de publication + KPI (3/3 cas d'usage)
  • make release-proof-notes: générer des notes de version orientées preuves + tag candidat
  • make all-tests : exécuter un inventaire de tests groupés complet
  • make reports-index: build target/reports/index.json (registre de rapports unifié)
  • make release-doctor : exécutez la suite de rapports sur l'état de préparation des instantanés/versions

VitteOS

  • make vitteos-doctor: imprimer les diagnostics de l'environnement VitteOS
  • make vitteos-quick: exécuter une boucle locale rapide (problèmes + domaine + orphelin + espace + arch-contrat + en-tête + ciblé + fumée)
  • make vitteos-ci : exécutez la chaîne VitteOS CI (scripts logiciels + porte bin + problèmes + domaine + orphelin + espace + arch-contrat + en-tête + ciblé + fumée + adr)
  • make vitteos-ci-strict : exécutez une chaîne VitteOS CI stricte (scripts stricts + porte bin + problèmes + domaine + orphelin + espace + arch-contrat + en-tête + ciblé + fumée + adr)

Conditionnement

  • make pkg-debian: construire l'installateur Debian .deb (PACKAGE_PROFILE=full, PKG_VERSION=2.1.1)
  • make pkg-debian-audit: audit généré du contenu Debian .deb et des fichiers les plus volumineux
  • make pkg-macos: construire le paquet d'installation macOS (PKG_VERSION=2.1.1)
  • make release-check: exécuter build + core-release-gate + ci-fast + ci-complétions + pkg build
