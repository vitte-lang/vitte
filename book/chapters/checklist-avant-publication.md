# Checklist Avant Publication

## Clarte

- Une section traite une idee principale.
- Aucune phrase mecanique ou interchangeable.
- Le vocabulaire reste stable.

## Exemples

- Cas nominal explicite.
- Cas limite reel explicite.
- Sortie attendue verifiable.

## Code

- Indentation uniforme dans les blocs `vit`.
- Lignes vides limitees aux separations utiles.
- Commentaires `//` courts et contextuels.

## Testabilite

- Garde visible sur entree invalide.
- Branche de decision explicable.
- Sortie finale tracable.

## QA finale

- Pas de repetition editoriale.
- Pas de phrase prefabriquee.
- Liens "voir aussi" coherents.
