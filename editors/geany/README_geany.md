# Geany Integration For Vitte

This directory provides Geany support for `.vit` and `.vitte` files.

## Files
- `filetypes.vitte.conf`: generated syntax, keywords, indentation, build menu.
- `snippets.vitte.conf`: language snippets (`[vitte]` and `[Vitte]` sections).
- `filetype_extensions.conf`: extension mapping for `*.vit` and `*.vitte`.
- `filetypes.common`: optional shared Geany defaults.
- `install_geany.sh`: installs/updates config in Geany user home(s).
- `uninstall_geany.sh`: removes Vitte Geany config from Geany user home(s).

## Install
1. Run `./editors/geany/install_geany.sh` (or `make install-geany`).
2. Restart Geany.
3. Open a `.vit` file and ensure filetype is `vitte`.

## Working Directory Mode
`VITTE_GEANY_WD_MODE` controls build menu working directory token:
- `file`: `%d` (directory of current file) [default]
- `project`: `%p` (Geany project directory)
- `current`: `%c` (Geany current working directory)

Example:

```bash
VITTE_GEANY_WD_MODE=project make install-geany
```

## Build Menu Commands
- File: Check, Build, Run, Test, Format, Parse.
- Project-aware: Project Check, Project Test.
- Workspace tools: Symbols, Workspace Symbols, Quickstart Check (Makefile), Core Language Gate (Makefile).

## Local Verification
- `make geany-lint`: checks generated Geany config and snapshots.
- `make geany-install-check`: validates local Geany install (filedefs, mappings, snippets).

## Uninstall
- `./editors/geany/uninstall_geany.sh` (or `make uninstall-geany`)

## Notes
- `filetypes.vitte.conf` is generated by `tools/generate_editor_highlights.py`.
- If you regenerate highlights (`make generate-highlights`), Geany config is refreshed automatically.
